# github-archiver

A minimal CLI tool to mirror GitHub repositories to another account or organization — branches, tags, description, and topics included.

No dependencies beyond `git`, `curl`, and `ssh`.

---

## Installation

### Homebrew (macOS and Linux)

```bash
brew install aymnms/tap/github-archiver
```

To update:
```bash
brew upgrade github-archiver
```

To uninstall:
```bash
brew uninstall github-archiver
```

### Manual

Clone the repo and run `./run.sh` directly.

---

## Prerequisites

- `git`
- `curl`
- `ssh` with a key configured for GitHub
- A GitHub Personal Access Token — see required permissions below

---

## Setup

```bash
./run.sh --setup
```

Prompts for:

| Field | Description |
|---|---|
| GitHub Personal Access Token | Used for HTTPS clone and GitHub API calls |
| Source username | GitHub account to clone repos from (defaults to `git config user.name`) |
| Destination org/user | GitHub account or org to archive repos to |
| Visibility | `public`, `private`, or `mirror` (copies source visibility) |

Saved to `.env` (git-ignored). Token is validated against the API before saving.

### Token permissions

**Classic token (`ghp_...`)** — scopes required:
- `repo` — full control of repositories (clone, create, update)
- `delete_repo` — only if using `--delete`

**Fine-grained token (`github_pat_...`)** — repository permissions:
- `Contents` → Read
- `Metadata` → Read (included by default)
- `Administration` → Write (required to create repositories)

> **Archiving to an organization?**  
> Your account must be a member of that organization with permission to create repositories. For fine-grained tokens, also grant **Organization permissions → Members → Read**.

## Usage

```bash
# Archive one or more repos
./run.sh <repo1> [repo2] [...]

# Archive then delete the source repos
./run.sh --delete <repo1> [repo2] [...]

# Reconfigure
./run.sh --setup

# Reconfigure then immediately archive
./run.sh --setup <repo1> [repo2] [...]
```

---

## How it works

```
run.sh
  ├── checks dependencies (git, curl, ssh)
  ├── loads .env
  └── calls push.sh <repos...>
        └── for each repo:
              1. git clone --mirror          → full copy (all branches, tags, refs)
              2. curl GET /repos/:source     → fetch description + visibility
              3. curl GET /repos/:source/topics → fetch topics
              4. curl GET/POST/PATCH :dest   → create or update destination repo
              5. curl PUT /topics            → sync topics
              6. git push --mirror           → push everything via SSH
              7. rm -rf local clone
```

Each repo runs in a subshell — a failure on one does not affect the others.

---

## Configuration

`.env` (generated by `--setup`):

```bash
GH_TOKEN="ghp_xxxxxxxxxxxxxxxxxxxx"   # Personal Access Token (scope: repo)
SOURCE_GITHUB_USER="my-username"      # Clone repos from this account
DEST_GITHUB_ORG="my-archive-org"      # Archive repos to this account/org
REPO_VISIBILITY="mirror"              # public | private | mirror
```

See `.env.example` for reference.
